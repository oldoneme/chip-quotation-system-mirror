# 统一编辑体验实施计划

## 🎯 业务需求
**目标**: 实现编辑报价单时回到原创建页面，而不是独立的编辑页面
**核心价值**: 统一用户体验，避免界面不一致造成的困惑

## 📊 影响范围分析
- **前端页面**: 6个报价创建页面需要支持编辑模式
- **后端API**: update_quote需要支持内容感知PDF重新生成
- **路由系统**: 删除临时QuoteEdit页面，修改编辑按钮跳转
- **数据流**: 报价单号保持不变，区分新建/编辑逻辑
